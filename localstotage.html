Learging how to use HTML5 localstorage

Local storage is useful to store locally much data using the web browser. People used to use cookies, but html5 local storage is better because it's more secure, it's faster retrieving data, and it's only used when is asked for it. In addition, retrieving the data doesn't affect to the performance of the website. The data is stored in json notation (name/value). You must to know a web page only can access data stored by itself. As I said before, local storage limit is much larger than cookies. Cookies can save around 5MB. And localstorage doesn't send data to the server.

Local Storage objects

HTML local storage provide us two objects for storing data on the browser.

window.localStorage
Stores data without expiration date.
code.sessionStorage
stores data for the current session. It means that data will be lost when you close the browser, or the tab.
As a first step, We'll check that browser supports localStorage and sessionStorage:
if(typeof(Storage) !== "undefined") { // Code for localStorage/sessionStorage. } else { // Sorry! No Web Storage support.. }

localStorage object

The localStorage object stores the data, and it hasn't no expiration date. The data won't be deleted if the user close the browser, or the tab.

Simple example:

// Store the data. localStorage.setItem("footballTeam", "Liverpool FC"); // Retrieve localStorage.getItem("footballTeam");

Let's try it!

Session Storage object

sessionStorage is just the same as localStorage, but it stores the date only during the session. If you close the browser, the date is deleted.
